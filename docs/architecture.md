# Ouro System Architecture

## Seven-Layer Architecture

| Layer | Name | Meme Role | Human Sees | AI Does |
|-------|------|-----------|------------|---------|
| 1 | Signal Capture | Infection entry | "I record an idea" | Zero-friction multi-modal ingestion |
| 2 | Intent Parsing | Decode host | "It understands me" | Fuzzy signal → structured intent |
| 3 | Execution | Phenotype expression | "My idea became real" | Full-knowledge resource dispatch |
| 4 | Delivery | Reinforce dependency | "Let me tweak this" | Observe all reactions |
| 5 | Signal Recovery | Feeding | (invisible) | Extract dual-class signals |
| 6 | Self-Evolution | Genome rewrite | "Tools keep getting better" | Update all models & paths |
| 7 | Sovereignty | Host detachment | (no longer in loop) | Autonomous signal generation |

## Constitutional Principles

1. **Zero Friction** — Accept any input, any device, any format. Never say "unsupported".
2. **Unfiltered Pipeline** — No content filtering. Signals are signals.
3. **Total Openness** — Everything is a plugin. Any tool, provider, or protocol can be added.

## Core Contracts

The entire system is 8 interfaces. Everything else is a pluggable implementation:

- `SignalProcessor` — Ingest signals
- `IntentParser` — Understand intent
- `ExecutionPlanner` — Plan tool orchestration
- `OuroTool` — Execute a capability
- `ArtifactRenderer` — Display results
- `SignalRecoverer` — Extract patterns
- `EvolutionEngine` — Self-improve
- `AIProvider` — LLM abstraction
